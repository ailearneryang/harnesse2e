## 1. 项目概述

### 1.1 项目目标

构建一个**单文件、自包含的 HTML5 跳跳棋（Chinese Checkers）网页游戏**，支持人机对战（人 vs AI），中文界面，交付标准对齐已有的 `snake.html`。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| **跳跳棋 / Chinese Checkers** | 六角星棋盘上的策略棋类游戏，目标是将己方全部棋子移至对角阵营 |
| **相邻移动** | 将棋子移至紧邻的空位（6 个方向之一），每次一格 |
| **跳跃** | 棋子越过一颗相邻棋子（任意方），到达其正对方向的空位 |
| **连续跳跃 (Chain Jump)** | 一次回合内同一棋子进行多次跳跃，不允许回跳已访问位置 |
| **阵营 / Home Zone** | 六角星的一个三角形尖端区域，每个阵营容纳 10 颗棋子 |
| **对角阵营** | 与己方初始阵营正对的阵营（即目标区域） |
| **贪心策略 AI** | 每步选择使棋子最接近目标区域的合法移动 |

### 1.3 项目范围

- **包含**: 棋盘渲染、棋子管理、移动规则引擎、贪心 AI 对手、游戏状态管理、回合制逻辑、交互提示、音效
- **不包含**: 在线多人模式、后端服务、用户账号系统、3~6 人对战、本地双人模式（首版仅人机对战）、存档/读档

### 1.4 参照先例

`src/snake.html`：388 行，单文件 HTML5 Canvas 游戏，中文 UI，Web Audio 音效，需求追踪注释。本项目对齐此交付标准。

## 2. 功能需求

### F001 — 六角星棋盘绘制

- **优先级:** P0
- **描述:** 在 HTML5 Canvas 上绘制标准六角星跳跳棋棋盘，包含 121 个有效位置，6 个三角形阵营区域（每区 10 个位置），中央六边形区域 61 个位置。
- **验收标准:**
  - **AC1:** Given 用户打开 `chinese_checkers.html` When 页面加载完成 Then Canvas 上显示完整的六角星形棋盘，121 个位置清晰可辨
  - **AC2:** Given 棋盘已渲染 When 用户观察棋盘 Then 6 个三角形阵营区域以不同底色标记，中央区域为中性色
  - **AC3:** Given 用户在不同分辨率设备上查看 When 窗口大小变化 Then 棋盘保持比例正确，自适应缩放，不溢出可视区域

### F002 — 棋盘坐标系统

- **优先级:** P0
- **描述:** 内部实现基于立方坐标（cube coordinates）或偏移坐标的六角网格坐标系统，支撑所有移动计算与像素渲染映射。
- **验收标准:**
  - **AC1:** Given 棋盘初始化 When 坐标系统建立 Then 121 个有效位置均有唯一坐标标识
  - **AC2:** Given 任意位置坐标 When 查询相邻位置 Then 正确返回最多 6 个相邻坐标（六方向）
  - **AC3:** Given 鼠标/触控点击事件 When 计算对应棋盘位置 Then 像素坐标准确映射到最近的有效棋盘位置点

### F003 — 棋子初始化与渲染

- **优先级:** P0
- **描述:** 游戏开始时，两方各 10 枚棋子分别放置在棋盘两个对角的三角形阵营内，双方棋子颜色不同（其余 4 个阵营为空）。
- **验收标准:**
  - **AC1:** Given 游戏初始化（2 人模式） When 棋子被放置 Then 玩家方 10 枚棋子放在一个三角区域，AI 方 10 枚棋子放在其正对的三角区域
  - **AC2:** Given 棋子已渲染 When 用户观察 Then 两方棋子颜色明显不同（如蓝色 vs 红色），棋子为圆形且大小适中
  - **AC3:** Given 棋子放置完成 When 用户对比棋盘 Then 每枚棋子精确居中在对应位置点上

### F004 — 棋子选中与合法落点提示

- **优先级:** P0
- **描述:** 当前回合玩家点击自己的棋子时，该棋子高亮显示，并展示所有合法目标位置（含相邻移动和跳跃）。
- **验收标准:**
  - **AC1:** Given 轮到玩家回合 When 玩家点击自己的一枚棋子 Then 该棋子以高亮效果（如发光边框或放大）显示
  - **AC2:** Given 棋子已选中 When 合法移动计算完成 Then 所有可到达的空位以半透明圆点或高亮标记显示
  - **AC3:** Given 棋子已选中 When 玩家点击另一枚自己的棋子 Then 取消当前选中，改为选中新棋子并显示其合法目标
  - **AC4:** Given 棋子已选中 When 玩家点击非合法目标的空白区域 Then 取消选中状态

### F005 — 相邻移动规则

- **优先级:** P0
- **描述:** 玩家可以将棋子移动到相邻的空位（最多 6 个方向之一）。
- **验收标准:**
  - **AC1:** Given 玩家选中了一颗己方棋子 When 玩家点击该棋子相邻的空位 Then 棋子移至该空位，回合结束，轮到 AI
  - **AC2:** Given 相邻位置已被其他棋子占据 When 计算合法目标 Then 该方向不显示为合法移动目标

### F006 — 跳跃规则

- **优先级:** P0
- **描述:** 棋子可以越过一颗紧邻的棋子（任意方），跳至其正对方向的空位。只能跳过紧邻的单个棋子（不能跳过空位或两个连续棋子）。
- **验收标准:**
  - **AC1:** Given 玩家选中棋子 A，A 在某方向上紧邻一颗棋子 B，B 正对方向的格子为空 When 玩家点击 B 正对方向的空位 Then 棋子 A 跳至该空位
  - **AC2:** Given B 正对方向的格子被占据或超出棋盘 When 计算跳跃目标 Then 该方向不标记为合法跳跃目标
  - **AC3:** Given 棋子位于棋盘边缘 When 计算合法移动 Then 不会产生越界的目标位置

### F007 — 连续跳跃（Chain Jump）

- **优先级:** P0
- **描述:** 跳跃后若仍有合法跳跃可执行，棋子可继续跳跃（不限次数），但不允许回跳至本次连跳中已访问过的位置。玩家可随时选择结束连续跳跃。
- **验收标准:**
  - **AC1:** Given 棋子完成一次跳跃后在新位置仍存在合法跳跃 When 系统计算可跳跃位置 Then 显示继续跳跃的合法目标，同时提供"结束回合"按钮/操作
  - **AC2:** Given 玩家点击下一个合法跳跃目标 When 跳跃执行 Then 棋子继续跳跃至该位置，且已访问位置不再可选
  - **AC3:** Given 玩家点击"结束回合"或非合法跳跃位置 When 操作执行 Then 连续跳跃结束，回合切换
  - **AC4:** Given 棋子正在连续跳跃中 When 系统计算可跳跃位置 Then 使用 visited set 排除已经过的位置，防止死循环

### F008 — 二人对战模式（人 vs AI）

- **优先级:** P0
- **描述:** 游戏支持两名玩家——人类玩家和 AI 对手，使用六角星棋盘的两个对角阵营，人类先手。
- **验收标准:**
  - **AC1:** Given 游戏启动 When 初始化完成 Then 两个对角阵营各放置 10 颗棋子（颜色不同），其余 4 个阵营为空
  - **AC2:** Given 游戏开始 When 第一回合 Then 人类玩家先手
  - **AC3:** Given 轮到玩家回合 When 玩家点击 AI 方棋子 Then 无任何响应（不可选中对方棋子）

### F009 — AI 对手（贪心策略）

- **优先级:** P0
- **描述:** AI 使用贪心策略——对所有己方棋子评估所有合法移动（含跳跃/连跳），选择使棋子最接近目标阵营（或使全局距离总和最小）的移动。
- **验收标准:**
  - **AC1:** Given 轮到 AI 回合 When AI 计算完成 Then AI 在 ≤2 秒内做出合法移动
  - **AC2:** Given AI 有多种合法移动 When AI 评估移动 Then 优先选择能使棋子（或全局）更接近目标阵营的移动
  - **AC3:** Given AI 可以跳跃或连跳 When 跳跃能比单步更快接近目标 Then AI 选择跳跃路径
  - **AC4:** Given AI 棋子已在目标区域 When AI 有其他棋子未到达目标 Then AI 优先移动未到达目标的棋子
  - **AC5:** Given AI 行动中 When 玩家观察 Then AI 移动有视觉动画展示，玩家在此期间不可操作

### F010 — 胜负判定

- **优先级:** P0
- **描述:** 当某一方的 10 颗棋子全部占据对角阵营的 10 个位置时，该方获胜，游戏结束。
- **验收标准:**
  - **AC1:** Given 玩家（或 AI）移动后 When 系统检测该方全部 10 颗棋子均在对角阵营内 Then 游戏结束，显示获胜方中文信息
  - **AC2:** Given 游戏结束 When 结果画面展示 Then 提供"重新开始"按钮
  - **AC3:** Given 游戏进行中 When 尚未有任何一方全部到达 Then 游戏继续

### F011 — 中文用户界面

- **优先级:** P0
- **描述:** 游戏所有文字均使用中文，包括标题、按钮、状态提示、胜负信息。
- **验收标准:**
  - **AC1:** Given 页面加载完成 When 用户查看界面 Then 标题显示"跳跳棋"
  - **AC2:** Given 游戏各状态 When 用户查看按钮和提示 Then 按钮为中文（"开始游戏"、"重新开始"、"结束回合"），状态提示为中文（"轮到你了"、"AI 思考中..."、"你赢了！"、"AI 获胜！"）

### F012 — 回合指示器

- **优先级:** P1
- **描述:** 界面清晰显示当前是哪方的回合，以文字和颜色双重标识。
- **验收标准:**
  - **AC1:** Given 游戏进行中 When 回合切换 Then 界面状态区显示当前回合方信息（如"轮到你了"或"AI 思考中..."）
  - **AC2:** Given 回合指示更新 When 用户观察 Then 文字颜色与对应方棋子颜色一致

### F013 — 棋子移动动画

- **优先级:** P1
- **描述:** 棋子移动（含跳跃）时有平滑的过渡动画，连跳时逐步展示跳跃路径。
- **验收标准:**
  - **AC1:** Given 棋子发生单步移动 When 系统执行移动 Then 棋子以平滑动画从起点移至终点，动画时长 ≤300ms
  - **AC2:** Given 棋子执行连跳 When 多次跳跃依次发生 Then 每跳一步动画 ≤200ms，逐跳播放
  - **AC3:** Given 动画播放中 When 用户尝试操作 Then 忽略用户输入，直到动画完成

### F014 — 音效反馈

- **优先级:** P2
- **描述:** 游戏关键操作伴有音效反馈，使用 Web Audio API 合成，无需外部音频文件。
- **验收标准:**
  - **AC1:** Given 棋子完成一次移动/跳跃 When 动画结束 Then 播放短促音效（≤200ms）
  - **AC2:** Given 游戏结束（任一方获胜） When 胜负判定触发 Then 播放胜利/失败音效（≤500ms）
  - **AC3:** Given 浏览器不支持 Web Audio API When 音效触发 Then 静默跳过，不影响游戏功能

### F015 — 响应式布局与触控支持

- **优先级:** P1
- **描述:** 游戏在桌面浏览器和移动端浏览器均可正常游玩，移动端支持触控操作。
- **验收标准:**
  - **AC1:** Given 用户在桌面浏览器（≥1024px 宽）打开游戏 When 页面加载 Then 棋盘居中显示，大小适配视口
  - **AC2:** Given 用户在移动端浏览器（≤768px 宽）打开游戏 When 页面加载 Then 棋盘自适应屏幕，触控点击正常工作，点击区域适当放大
  - **AC3:** Given 用户旋转设备 When 视口尺寸变化 Then 棋盘重新适配，游戏状态不丢失

### F016 — 游戏状态管理

- **优先级:** P0
- **描述:** 游戏具备完整的状态机：IDLE → PLAYING → GAME_OVER，支持开始和重新开始。
- **验收标准:**
  - **AC1:** Given 页面刚加载 When 状态为 IDLE Then 显示"开始游戏"按钮，棋盘为初始状态
  - **AC2:** Given 用户点击"开始游戏" When 状态切换到 PLAYING Then 人类玩家可操作，AI 在轮到时自动行动
  - **AC3:** Given 游戏结束 When 状态切换到 GAME_OVER Then 显示结果和"重新开始"按钮
  - **AC4:** Given 用户点击"重新开始" When 触发重置 Then 游戏回到初始状态，重新开始

## 3. 非功能需求

### NFR001 — 单文件零依赖

- **优先级:** P0
- **描述:** 交付物为单个 HTML 文件 `src/chinese_checkers.html`，包含所有 HTML、CSS、JavaScript，不引用任何外部 CDN、库、字体、图片或音频文件。
- **量化指标:** 外部网络请求数 = 0
- **验收标准:**
  - Given 将文件复制到无网络环境 When 用浏览器打开 Then 游戏完整运行，无任何资源加载失败

### NFR002 — HTML5 Canvas 渲染

- **优先级:** P0
- **描述:** 棋盘和棋子使用 HTML5 Canvas 2D API 绘制，不使用 DOM 元素。
- **量化指标:** 渲染帧率 ≥30fps（含动画期间）
- **验收标准:**
  - Given 游戏运行中 When 棋盘重绘 Then 使用 Canvas 2D Context 渲染，无 DOM 元素用于棋盘/棋子

### NFR003 — 需求追踪注释

- **优先级:** P0
- **描述:** 代码中关键模块和函数须嵌入需求编号注释（如 `// F001: 棋盘绘制`），确保需求到代码的可追踪性。
- **量化指标:** 每个 P0 功能需求至少有 1 处代码注释引用
- **验收标准:**
  - Given 审查源代码 When 搜索需求编号 Then 每个 P0 需求编号（F001~F011, F016）至少出现 1 次

### NFR004 — WCAG AA 色彩对比度

- **优先级:** P1
- **描述:** 关键 UI 元素（文字、棋子、按钮）与背景的颜色对比度满足 WCAG AA 标准。
- **量化指标:** 文字对比度 ≥4.5:1，大型图形元素 ≥3:1
- **验收标准:**
  - Given 游戏运行中 When 检查颜色对比度 Then 文字与背景对比度 ≥4.5:1，棋子与棋盘对比度 ≥3:1

### NFR005 — 文件体积控制

- **优先级:** P1
- **描述:** 单文件 HTML 总体积控制在合理范围。
- **量化指标:** 文件总行数 ≤1000 行
- **验收标准:**
  - Given 交付文件 When 统计行数 Then 总行数 ≤1000 行

### NFR006 — 浏览器兼容性

- **优先级:** P1
- **描述:** 游戏在主流现代浏览器上正常运行。
- **量化指标:** 支持 Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **验收标准:**
  - Given 使用上述任一浏览器打开文件 When 游戏加载 Then 无 JavaScript 错误，所有 P0 功能正常

### NFR007 — AI 响应时间

- **优先级:** P0
- **描述:** AI 回合的计算时间不应造成用户可感知的长时间卡顿。
- **量化指标:** AI 单步决策 ≤2 秒；合法移动计算 ≤100ms
- **验收标准:**
  - Given 轮到 AI 行动 When AI 开始计算 Then 从回合开始到棋子移动完成 ≤2 秒

### NFR008 — 安全性

- **优先级:** P0
- **描述:** 游戏不收集用户数据，不发起网络请求，无 XSS/注入风险。
- **量化指标:** 外部请求 = 0；不使用 eval/innerHTML 动态执行用户输入
- **验收标准:**
  - Given 安全审查代码 When 检查网络与数据处理 Then 无外部请求、无 eval、无未过滤的 innerHTML

## 7. 风险清单

| 编号 | 风险描述 | 影响 | 缓解措施 |
|------|---------|------|---------|
| R001 | 六角星坐标系实现复杂，可能导致棋盘绘制和移动计算错误 | 高 | 设计阶段充分推演坐标方案（推荐立方坐标系），编写验证逻辑 |
| R002 | 连续跳跃检测可能产生死循环 | 高 | 强制使用 visited set 记录已访问位置，限制最大跳跃深度 |
| R003 | 单文件体积可能超标 | 中 | 代码精简，复用绘制函数，控制在 1000 行以内 |
| R004 | 移动端六角星格子过小，触控困难 | 中 | 放大触控区域，增加视觉选中反馈 |
| R005 | AI 贪心策略可能导致棋子陷入局部最优或循环移动 | 中 | AI 添加禁忌列表（近 N 步不重复），或加入随机扰动 |
| R006 | 原始需求仅含标题，交付物可能与用户期望不符 | 中 | 本文档已做合理推断并记录所有假设 |
