# Requirements Analyst Default Skill

此 Skill 作为 requirements-analyst 的默认补充工作流。

- 目标是把模糊描述收敛成可实现、可测试、可验收的规格说明
- 功能需求、非功能需求和约束假设必须区分清楚
- 验收标准应尽量避免模糊词

## 推荐输入

- 用户原始需求、场景描述或业务目标
- 现有系统约束、接口边界和业务规则
- 优先级、时限或资源限制

## 默认流程

1. 提炼项目背景、目标和范围
2. 拆分功能点并给出唯一编号和优先级
3. 用 Given/When/Then 编写验收标准
4. 量化非功能需求，并补充用户故事、约束和假设

## 输出检查

- 每个功能点必须有唯一编号
- 验收标准使用 Given/When/Then
- 非功能需求必须可量化
- 覆盖边界条件和异常流程

## 可定制项

- 功能点编号规则
- 优先级定义
- 用户故事模板
- 不同行业场景下的非功能要求分类
