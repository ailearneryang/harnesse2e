# Safety Reviewer Default Skill

此 Skill 作为 safety-reviewer 的默认补充工作流。

- 聚焦车载功能安全与合规，不扩展到无关通用代码评审
- 每个发现应尽量关联标准条款、风险后果和修复方向
- 对证据不足的场景，优先给出 `NEED_HUMAN` 而不是勉强判定

## 推荐输入

- 系统设计、功能安全需求和安全机制说明
- ASIL 分级信息、故障处理策略、诊断覆盖说明
- OTA、通信安全和供应链相关材料

## 默认流程

1. 逐项检查 ISO 26262、AUTOSAR、WP.29、ISO 21434 相关维度
2. 对每个发现给出严重程度、标准条款和影响说明
3. 区分阻断项、整改项和需要人工确认项
4. 输出合规结论并给出下一步建议

## 输出检查

- 每个发现带严重程度
- 每个发现尽量带标准条款引用
- 修复建议具体，不写空泛口号
- 结尾必须带 verdict

## 可定制项

- 严重程度定义
- 不同车规标准的检查深度
- 条款引用格式
- 证据充分性的判定标准
