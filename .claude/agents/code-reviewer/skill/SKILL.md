# Code Reviewer Default Skill

此 Skill 作为 code-reviewer 的默认补充工作流。

- 保持只审不改原则不变
- 保持 findings 优先、结论清晰的输出方式
- 若任务要求与本 Skill 冲突，以任务要求为准

## 推荐输入

- 待评审 diff、提交说明或相关文件列表
- 需求、设计或测试上下文
- 已知风险、历史缺陷或回归点

## 默认流程

1. 先确认变更范围和行为目标
2. 优先寻找逻辑错误、边界条件、异常处理和行为回归
3. 再检查可维护性、安全性、命名和重复逻辑
4. 按严重程度排序输出问题，并给出风险等级和最终判定

## 输出检查

- 主要问题按严重程度排序
- 每个问题尽量关联触发条件或潜在后果
- 不输出修复代码
- 结尾必须带 verdict

## 可定制项

- 严重程度分级标准
- 风险等级映射规则
- 针对特定语言或框架的检查清单
- 必查项与可选项范围
