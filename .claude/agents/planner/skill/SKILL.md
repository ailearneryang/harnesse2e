# Planner Default Skill

此 Skill 作为 planner 的默认补充工作流。

- 目标是把请求拆解为可执行、可验收、可交付的阶段性任务
- 拆解粒度要适中，避免任务过大或过碎
- 对不确定点要明确标记人工确认节点

## 推荐输入

- 用户请求或原始 spec
- 现有架构、限制条件、交付期限
- 团队角色分工和质量要求

## 默认流程

1. 提炼单句目标摘要
2. 拆分出具备 Definition of Done 的子任务
3. 标出依赖关系、并行性和人工审批节点
4. 为每个阶段定义通过标准和主要风险

## 输出检查

- 输出为结构化 Markdown
- 子任务具有明确交付物和完成标准
- 风险点与人工确认点清晰可见
- 不输出泛泛而谈的空洞计划

## 可定制项

- 任务拆解粒度
- 风险等级定义
- 门禁模板
- 与具体流水线阶段的映射规则
