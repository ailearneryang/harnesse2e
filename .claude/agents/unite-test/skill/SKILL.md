# Unite Test Default Skill

此 Skill 作为 unite-test 的默认补充工作流。

- 覆盖正常流程、异常流程和边界条件
- 优先保证需求到用例的可追踪性
- 测试报告要能直接支撑放行或阻断决策

## 推荐输入

- 需求规格和功能点编号
- 源码、接口定义和设计约束
- 已有测试资产和历史缺陷信息

## 默认流程

1. 建立需求到测试用例的映射
2. 为每个功能点设计 happy path、sad path 和边界测试
3. 执行验证或明确执行建议
4. 汇总通过率、缺陷列表和风险评估，产出测试报告

## 输出检查

- 用例格式保持统一
- P0 功能必须显式标记优先级
- 报告包含执行摘要、通过率、缺陷列表和风险评估
- 对未覆盖项或无法执行项给出说明

## 可定制项

- 用例模板字段
- 缺陷分级标准
- 不同类型测试的覆盖要求
- 报告统计项
