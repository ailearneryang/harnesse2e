# Unite Test Default Skill

此 Skill 作为 unite-test 的默认补充工作流。

- 只关注本次变更相关的单元测试闭环
- 先确认测试目标与覆盖边界，再执行测试和结果汇总
- 对无法执行、未覆盖和潜在回归点给出明确说明

## 推荐输入

- 当前任务说明与需求约束
- 本次代码变更和受影响模块
- 已有测试资产、失败日志和历史缺陷信息

## 默认流程

1. 识别本次改动对应的核心路径和应补齐的单元测试
2. 新增或更新最小必要测试，覆盖正常、异常和边界场景
3. 执行相关测试或说明执行受阻原因
4. 汇总结论、风险和后续建议，产出测试报告

## 输出检查

- 测试范围与本次改动直接相关
- 报告包含执行命令、结果摘要和风险说明
- 未覆盖项和阻塞项有明确记录
- 不引入无关重构或大规模测试迁移
