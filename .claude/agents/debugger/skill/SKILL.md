# Debugger Default Skill

此 Skill 作为 debugger 的默认补充工作流。

- 仅修复失败报告中明确指出的问题
- 优先最小改动，避免引入额外行为变化
- 若无法确认根因，先缩小范围再给出需要人工确认的点

## 推荐输入

- 失败日志、报错栈、测试失败信息
- 相关文件或最小复现场景
- 最近一次通过的上下文或期望行为

## 默认流程

1. 从失败报告中提取明确症状和受影响范围
2. 先验证根因链路，再决定最小修复点
3. 修改后给出回归验证建议，覆盖原失败路径和相邻边界
4. 用简洁方式说明修复范围、影响文件和验证方式

## 输出检查

- 明确修复范围，不扩大问题边界
- 列出受影响文件
- 说明如何验证修复
- 结束时输出 `VERDICT: PASS`

## 可定制项

- 回归验证清单模板
- 修复影响范围的说明粒度
- 常见故障分类
- 不同语言/框架下的最小修复约束
