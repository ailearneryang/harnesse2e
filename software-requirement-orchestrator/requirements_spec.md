## 1 范围与目的

本文档定义越野信息算法集成到集成座舱控制器（ICC）的软件需求，覆盖车辆倾斜角、车辆俯仰角、海拔高度、大气压力、指南针方向/角度和清零功能。  
本文档用于指导需求评审、架构设计、开发联调和测试验收，适用于复用 Android 标准 IMU/GPS 接口以及车身 CAN 气压信号的 ICC 项目。  
文档中的未闭合参数统一标记为 **[待确认]**，仅对开发、联调、测试必需的信息做 **[工程化补足]**。

### 1.1 目标

- 明确越野信息算法在 ICC 上的功能范围、外部边界和可交付结果。
- 将上传规范中的场景化描述拆解为可独立实现、独立验收的软件需求。
- 汇总待确认项、外部依赖和工程约束，供后续设计、开发和测试阶段闭环。

### 1.2 非目标

- 不描述算法实现公式、滤波器实现细节、类设计和线程模型。
- 不展开 CAN 报文字段级位定义，相关内容以 DBC 为准。
- 不替代缺失的《安卓接口需求说明》或算法设计说明。

## 2 系统背景与边界

### 2.1 系统背景

越野信息算法集成于 ICC，由 ICC 复用内置 IMU、Android 定位信息、整车车速和 EMS 大气压力信号，计算越野信息并在中控/仪表显示，同时通过 CAN 0x4F0 报文对外发送给 HUD、水深模块等使用方。

### 2.2 系统职责

- 接收并校验 IMU、GPS、车速、气压和清零指令输入。
- 计算并输出倾斜角、俯仰角、海拔、大气压力、指南针方向/角度及其有效标志。
- 管理清零参数、车型姿态补偿参数和指南针记忆值。
- 在异常、失效和恢复场景下输出一致、可诊断的结果。

### 2.3 系统边界

| 项目 | 本系统责任 | 非本系统责任 |
|------|------------|--------------|
| IMU/GPS 数据获取 | 订阅 Android 标准接口并使用输入数据 | Android Framework/驱动层采集与底层校准 |
| 气压输入 | 接收并使用 EMS 气压信号 | EMS 信号生成、总线传输质量保证 |
| 越野信息显示 | 向 ICC 显示链路提供计算结果和有效状态 | UI 视觉设计与界面资产制作 |
| CAN 0x4F0 输出 | 周期发送越野信息结果 | DBC 定义、总线仲裁与对端解析实现 |
| 清零功能 | 响应清零触发、执行条件判断、持久化结果 | 工厂/售后配置流程与权限管控策略最终定义 |
| 车型姿态适配 | 加载车型配置并做姿态补偿 | 配置文件生成、版本发布与下发平台 |

## 4 术语与缩略语

| 术语/缩写 | 定义 |
|-----------|------|
| ICC | 集成座舱控制器，越野信息算法运行平台 |

| GPS | 卫星定位信息来源，用于海拔和航向辅助 |

| Pitch Angle | 车辆前后俯仰角 |
