## 1. 项目概述

### 1.1 项目背景
开发一个经典的贪吃蛇（Snake）小游戏，供用户在浏览器中游玩。贪吃蛇是一款广为人知的休闲游戏，玩家控制一条蛇在有限的区域内移动，吃掉食物使蛇身变长，同时避免撞墙或撞到自身。

### 1.2 项目目标
- 实现一个功能完整、体验流畅的贪吃蛇网页游戏
- 支持键盘操作和移动端触屏操作
- 提供计分系统和游戏状态管理

### 1.3 项目范围
- **包含**: 游戏核心逻辑、UI 界面、计分系统、游戏状态管理、响应式布局
- **不包含**: 多人在线模式、后端服务、用户账号系统、排行榜持久化存储

## 2. 功能需求

### F001: 游戏画布渲染
- **描述**: 使用 HTML5 Canvas 渲染一个固定大小的网格化游戏区域，作为贪吃蛇的活动场地
- **优先级**: P0
- **验收标准(AC)**:
  - Given 用户打开游戏页面 When 页面加载完成 Then 显示一个 20×20 的网格游戏画布，画布居中显示，背景与网格线清晰可见
  - Given 游戏画布已渲染 When 用户在不同分辨率设备上查看 Then 画布保持比例正确，不溢出可视区域

### F002: 蛇的初始化与渲染
- **描述**: 游戏开始时，在画布中央生成一条初始长度为 3 个单元格的蛇，蛇头与蛇身颜色有区分
- **优先级**: P0
- **验收标准(AC)**:
  - Given 游戏初始化 When 蛇被生成 Then 蛇出现在画布中央区域，初始长度为 3 格，默认朝右移动
  - Given 蛇已渲染 When 用户观察蛇 Then 蛇头颜色与蛇身颜色不同，可明确区分方向

### F003: 蛇的移动
- **描述**: 蛇按照固定时间间隔自动向当前方向移动，每次移动一个网格单元
- **优先级**: P0
- **验收标准(AC)**:
  - Given 游戏处于进行中状态 When 经过一个游戏帧间隔(150ms) Then 蛇向当前方向移动一格
  - Given 蛇正在移动 When 未接收到方向改变输入 Then 蛇保持当前方向持续移动

### F004: 键盘方向控制
- **描述**: 玩家通过键盘方向键（↑↓←→）或 WASD 控制蛇的移动方向
- **优先级**: P0
- **验收标准(AC)**:
  - Given 游戏进行中 When 玩家按下方向键/WASD Then 蛇在下一帧改变为对应方向移动
  - Given 蛇当前向右移动 When 玩家按下左方向键 Then 忽略该输入（禁止 180° 反向移动）
  - Given 玩家在一帧内连续按下多个方向键 When 下一帧到来 Then 只响应第一次有效的方向改变

### F005: 触屏滑动控制
- **描述**: 移动端用户可通过在游戏区域滑动手指来控制蛇的方向
- **优先级**: P1
- **验收标准(AC)**:
  - Given 用户在移动设备上游玩 When 在游戏区域向上/下/左/右滑动 Then 蛇改变为对应方向移动
  - Given 用户滑动距离小于 30px When 触摸结束 Then 不触发方向改变（防止误触）

### F006: 食物生成
- **描述**: 在游戏区域内随机生成食物，食物不能出现在蛇身占据的位置上
- **优先级**: P0
- **验收标准(AC)**:
  - Given 游戏开始或食物被吃掉 When 需要生成新食物 Then 在空闲格子中随机选取一个位置生成食物
  - Given 蛇身占据了大量格子 When 生成食物 Then 食物必定出现在非蛇身的空闲格子上
  - Given 食物已生成 When 用户观察 Then 食物以醒目颜色（红色）在网格中显示

### F007: 吃食物与蛇身增长
- **描述**: 蛇头移动到食物位置时，蛇身长度增加 1 格，同时触发新食物生成
- **优先级**: P0
- **验收标准(AC)**:
  - Given 蛇正在移动 When 蛇头到达食物所在的格子 Then 蛇身长度增加 1 格，食物消失，新食物在随机空闲位置生成
  - Given 蛇吃到食物 When 蛇身增长 Then 增长部分添加在蛇尾，不影响蛇头移动方向

### F008: 碰撞检测 — 撞墙
- **描述**: 蛇头超出游戏区域边界时，判定游戏结束
- **优先级**: P0
- **验收标准(AC)**:
  - Given 蛇正在向上移动 When 蛇头到达画布顶部边界并继续移动 Then 触发游戏结束
  - Given 蛇正在向左移动 When 蛇头到达画布左边界并继续移动 Then 触发游戏结束

### F009: 碰撞检测 — 撞自身
- **描述**: 蛇头移动到蛇身任意部分占据的格子时，判定游戏结束
- **优先级**: P0
- **验收标准(AC)**:
  - Given 蛇长度 ≥ 5 When 蛇头移动到蛇身某节占据的格子 Then 触发游戏结束
  - Given 蛇长度 < 4 When 蛇正常移动 Then 不会因自身碰撞结束（长度不足以碰到自己）

### F010: 计分系统
- **描述**: 每吃一个食物得 10 分，当前分数实时显示在游戏界面上方
- **优先级**: P0
- **验收标准(AC)**:
  - Given 游戏开始 When 初始状态 Then 分数显示为 0
  - Given 蛇吃到一个食物 When 分数更新 Then 分数增加 10 分，UI 实时更新显示
  - Given 游戏进行中 When 用户查看分数区域 Then 当前分数清晰可见

### F011: 最高分记录
- **描述**: 记录玩家历史最高分，存储在浏览器 localStorage 中，跨会话保留
- **优先级**: P1
- **验收标准(AC)**:
  - Given 游戏结束 When 当前分数 > 历史最高分 Then 更新最高分并存入 localStorage
  - Given 用户重新打开页面 When 页面加载 Then 从 localStorage 读取并显示历史最高分
  - Given 无历史记录 When 首次游戏 Then 最高分默认显示为 0

### F012: 游戏状态管理（开始/暂停/结束/重新开始）
- **描述**: 提供完整的游戏生命周期管理，包括开始、暂停、恢复、结束和重新开始
- **优先级**: P0
- **验收标准(AC)**:
  - Given 页面加载完成 When 用户看到游戏界面 Then 显示"开始游戏"按钮或提示，蛇处于静止状态
  - Given 游戏进行中 When 用户按下空格键 Then 游戏暂停，画面冻结，显示"已暂停"提示
  - Given 游戏已暂停 When 用户再次按下空格键 Then 游戏恢复运行
  - Given 游戏结束 When 用户看到结束画面 Then 显示最终得分和"重新开始"按钮
  - Given 游戏结束画面 When 用户点击"重新开始" Then 游戏重置为初始状态并重新开始

### F013: 速度递增机制
- **描述**: 随着蛇吃到更多食物，游戏速度逐渐加快，增加难度
- **优先级**: P2
- **验收标准(AC)**:
  - Given 初始游戏帧间隔为 150ms When 蛇每吃 5 个食物 Then 帧间隔减少 10ms
  - Given 帧间隔持续减小 When 帧间隔到达 50ms Then 不再继续减小（设定速度下限）

### F014: 游戏音效
- **描述**: 为吃食物和游戏结束提供简短音效反馈
- **优先级**: P3
- **验收标准(AC)**:
  - Given 蛇吃到食物 When 碰撞发生 Then 播放一个短促的音效（≤ 200ms）
  - Given 游戏结束 When 碰撞墙壁或自身 Then 播放游戏结束音效（≤ 500ms）
  - Given 用户设备不支持音频或用户静音 When 音效触发 Then 不报错，静默跳过

## 3. 非功能需求

### NFR001: 性能
- 游戏帧渲染时间 ≤ 16ms（确保 60fps 流畅渲染）
- 页面首次加载到可交互时间 ≤ 2s（在 4G 网络条件下）
- 游戏核心逻辑（移动、碰撞检测）单次执行耗时 ≤ 1ms

### NFR002: 兼容性
- 浏览器支持: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 移动端支持: iOS Safari 14+, Android Chrome 90+
- 屏幕分辨率: 最小支持 320×568（iPhone SE），最大支持 2560×1440

### NFR003: 可用性
- 游戏操作说明在界面上可见，新用户无需额外指引即可开始游戏
- 色彩对比度满足 WCAG 2.0 AA 标准（对比度 ≥ 4.5:1）

### NFR004: 代码质量
- 单个 HTML 文件交付（HTML + CSS + JavaScript 内联），便于分享和部署
- 代码总量 ≤ 500 行
- 无外部依赖，不使用任何第三方库

### NFR005: 安全性
- 不收集任何用户数据
- localStorage 仅存储最高分数值（整数类型）
- 无网络请求

## 5. 约束与假设

### 5.1 约束
- **技术约束**: 使用纯前端技术实现（HTML5 Canvas + CSS + Vanilla JavaScript），单文件交付
- **交付约束**: 不依赖任何构建工具、包管理器或第三方 CDN
- **运行环境**: 仅在现代浏览器中运行，不兼容 IE

### 5.2 假设
- 用户设备支持 HTML5 Canvas API
- 用户使用标准尺寸键盘（含方向键）或触屏设备
- localStorage 功能可用（非隐私模式下）
- 用户对贪吃蛇基本玩法有基础认知

### 5.3 优先级总结

| 优先级 | 功能点 | 说明 |
|--------|--------|------|
| **P0** | F001~F004, F006~F010, F012 | 核心游戏玩法，必须实现 |
| **P1** | F005, F011 | 增强体验，推荐实现 |
| **P2** | F013 | 进阶体验，按时间余量实现 |
| **P3** | F014 | 锦上添花，可选实现 |

  - Given 游戏结束 When 用户看到结束画面 Then 显示最终得分和"重新开始"按钮
  - Given 游戏结束画面 When 用户点击"重新开始" Then 游戏重置为初始状态并重新开始

### F013: 速度递增机制
- **描述**: 随着蛇吃到更多食物，游戏速度逐渐加快，增加难度
- **优先级**: P2
- **验收标准(AC)**:
  - Given 初始游戏帧间隔为 150ms When 蛇每吃 5 个食物 Then 帧间隔减少 10ms
  - Given 帧间隔持续减小 When 帧间隔到达 50ms Then 不再继续减小（设定速度下限）

### F014: 游戏音效
- **描述**: 为吃食物和游戏结束提供简短音效反馈
- **优先级**: P3
- **验收标准(AC)**:
  - Given 蛇吃到食物 When 碰撞发生 Then 播放一个短促的音效（≤ 200ms）
  - Given 游戏结束 When 碰撞墙壁或自身 Then 播放游戏结束音效（≤ 500ms）
  - Given 用户设备不支持音频或用户静音 When 音效触发 Then 不报错，静默跳过

## 3. 非功能需求

### NFR001: 性能
- 游戏帧渲染时间 ≤ 16ms（确保 60fps 流畅渲染）
- 页面首次加载到可交互时间 ≤ 2s（在 4G 网络条件下）
- 游戏核心逻辑（移动、碰撞检测）单次执行耗时 ≤ 1ms

### NFR002: 兼容性
- 浏览器支持: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 移动端支持: iOS Safari 14+, Android Chrome 90+
- 屏幕分辨率: 最小支持 320×568（iPhone SE），最大支持 2560×1440

### NFR003: 可用性
- 游戏操作说明在界面上可见，新用户无需额外指引即可开始游戏
- 色彩对比度满足 WCAG 2.0 AA 标准（对比度 ≥ 4.5:1）

### NFR004: 代码质量
- 单个 HTML 文件交付（HTML + CSS + JavaScript 内联），便于分享和部署
- 代码总量 ≤ 500 行
- 无外部依赖，不使用任何第三方库

### NFR005: 安全性
- 不收集任何用户数据
- localStorage 仅存储最高分数值（整数类型）
- 无网络请求

## 5. 约束与假设

### 5.1 约束
- **技术约束**: 使用纯前端技术实现（HTML5 Canvas + CSS + Vanilla JavaScript），单文件交付
- **交付约束**: 不依赖任何构建工具、包管理器或第三方 CDN
- **运行环境**: 仅在现代浏览器中运行，不兼容 IE

### 5.2 假设
- 用户设备支持 HTML5 Canvas API
- 用户使用标准尺寸键盘（含方向键）或触屏设备
- localStorage 功能可用（非隐私模式下）
- 用户对贪吃蛇基本玩法有基础认知

### 5.3 优先级总结

| 优先级 | 功能点 | 说明 |
|--------|--------|------|
| **P0** | F001~F004, F006~F010, F012 | 核心游戏玩法，必须实现 |
| **P1** | F005, F011 | 增强体验，推荐实现 |
| **P2** | F013 | 进阶体验，按时间余量实现 |
| **P3** | F014 | 锦上添花，可选实现 |
